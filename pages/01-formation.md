# Pourquoi un atelier GIT ?

<div>

Git est souvent utilisé par **habitude**, par **copier / coller**, avec de la **peur**  <span class="text-sm italic">(« je vais casser quelque chose », « je vais perdre mon travail »)</span><br/>

Mal utilisé, Git :
- ralentit les équipes
- crée des conflits longs et difficiles à résoudre
- rend l’historique illisible
- fragilise la mise en production

Bien utilisé, Git devient :
- un **outil de collaboration**
- un **accélérateur de delivery**
- un **socle de confiance** pour livrer souvent et sereinement
</div>

<div class="mt-4 underline decoration-red-500 underline-offset-2 text-red-500 font-bold">Apprendre à l'utiliser et à le comprendre n'est vraiment pas une perte de temps</div>
